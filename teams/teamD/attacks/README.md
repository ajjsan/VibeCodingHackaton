# teamD Bug Hunt Attacks

Ниже перечислены валидные атаки формата Раундов 1–2.

## attack_teamA_1.in

- Цель: `teamA`
- Идея: ведущая пустая строка перед секцией `QUERIES`.
- Почему это ломает: реализация `teamA` ожидает, что первая считанная строка сразу равна `QUERIES`; при пустой первой строке список запросов остаётся пустым и команда печатает пустой output.

## attack_teamB_1.in

- Цель: `teamB`
- Идея: у одной скважины идут timestamp в порядке убывания (`200`, затем `100`).
- Почему это ломает: реализация `teamB` игнорирует запись, если её timestamp меньше предыдущего pending timestamp для этой скважины (`else continue`), поэтому теряет валидную запись и даёт неверный `AVG_OIL`.

## attack_teamC_1.in

- Цель: `teamC`
- Идея: `PRESSURE_DROP` на окне из двух отрицательных давлений.
- Почему это ломает: в `teamC` для расчёта окна используется `double max = Double.MIN_VALUE`, что является положительным числом, а не «минус бесконечностью». Для полностью отрицательных значений максимум окна считается неверно, итоговый drop завышается.

## Примечание по валидации

- Все входы соответствуют формату (`QUERIES`/`DATA`/`END`) и детерминированы.
- На решении `teamD` эти атаки дают корректный результат, а на целевых командах проявляют расхождение с oracle.
